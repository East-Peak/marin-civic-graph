# Data Layout

This directory is reserved for collected artifacts and derived outputs.

The intended flow is:

1. capture raw source artifacts
2. derive extracted machine-readable outputs
3. promote normalized records into graph-ready candidate objects

Subdirectories:

- `raw/`
- `extracted/`
- `normalized/`

Conventions for naming and storage live in [Artifact Conventions](../docs/artifact-conventions.md).

Until implementation begins, these directories mainly serve as stable placeholders so the first collection pass lands in a predictable layout.
